# MuiAD PRD v1

> 产品需求文档 · 第一版
> 最后更新：2026-04-14

## 1. 产品定位

**一句话**：MCP-first、Self-hosted、AI 驱动的去中心化开发者推广网络。

**目标用户**：独立开发者 / Indie Hacker，尤其是那些手里有多个产品、但不想花精力做推广的人。

**核心价值**：让开发者在写代码的时候，顺便就让 AI 把推广的事办了。

## 2. 用户画像

| 属性 | 描述 |
|------|------|
| 角色 | 独立开发者，有 1-10 个产品 |
| 技能 | 熟悉 TypeScript，会用 AI 编程工具（Cursor / Claude / Windsurf） |
| 痛点 | 能写代码但不想做推广；产品做好了没人知道；AdSense 申请不下来 |
| 期望 | 零推广操作，AI 全自动；数据在自己手里（self-hosted）；免费或极低成本 |

## 3. 使用模式

### 3.1 孤岛模式

不连接任何网络。将自己的多个产品注册到同一个 MuiAD 实例中，产品之间互相展示广告。

适用场景：有多个产品的开发者，想盘活自己的产品矩阵。

### 3.2 节点模式

部署自己的 MuiAD 实例，连接到公共网络（如 all-mui-ad），与其他开发者的节点互联。

适用场景：想要流量互换、扩大推广覆盖面的开发者。

### 3.3 免部署模式

不部署自己的实例，直接使用公共节点 all-mui-ad 的服务。

适用场景：不想折腾基础设施、想快速上手的开发者。

## 4. 核心功能

### 4.1 MCP Server（交互层）

MCP 是 MuiAD 的主要交互方式。AI Agent 通过 MCP Tool 操作 MuiAD。

#### 4.1.1 广告位管理

| Tool | 说明 |
|------|------|
| `muiad_create_zone` | 创建广告位。输入：名称、尺寸、所属站点。输出：zone_id + 嵌入代码片段 |
| `muiad_list_zones` | 列出当前实例的所有广告位及其状态 |
| `muiad_get_zone_stats` | 查看指定广告位的展示量、点击量、CTR |

#### 4.1.2 产品与广告管理

| Tool | 说明 |
|------|------|
| `muiad_register_product` | 注册产品。输入：URL（AI 自动抓取分析）或手动填写名称、描述、分类。输出：product_id |
| `muiad_create_ad` | 创建广告。输入：product_id、目标 zone_id、广告类型。AI 自动生成物料（banner 图、文案）。输出：ad_id |
| `muiad_list_ads` | 列出所有广告及其投放状态 |

#### 4.1.3 网络操作（MVP-3）

| Tool | 说明 |
|------|------|
| `muiad_connect_network` | 连接到一个网络节点（输入节点 URL） |
| `muiad_scan_available_zones` | 扫描已连接网络中的可用广告位 |
| `muiad_submit_ad_to_zone` | 向指定节点的指定广告位提交广告 |

#### 4.1.4 AI 推广（MVP-2）

| Tool | 说明 |
|------|------|
| `muiad_auto_promote` | 一键推广。输入：product_id + 预算。AI 自动完成扫描→匹配→生成物料→投放全流程 |
| `muiad_optimize_campaign` | AI 分析现有投放数据，自动优化（调整权重、替换低效物料、生成新版本） |

#### 4.1.5 数据与结算（MVP-1 / MVP-4）

| Tool | 说明 |
|------|------|
| `muiad_get_stats` | 查看推广效果（按时间范围、按产品、按广告位） |
| `muiad_get_credits` | 查看积分余额 |
| `muiad_list_conversions` | 查看转化明细（来源、事件类型、时间） |

### 4.2 广告投放引擎

- 根据广告位配置（尺寸、位置、受众标签）选择合适的广告
- 支持权重轮播、频次控制（同一用户 N 小时内不重复展示）
- 无可用广告时返回空或展示默认内容
- 记录每次展示（impression）和点击（click）

### 4.3 归因追踪（MVP-1）

- 广告链接自动附加归因参数（ref、campaign、creative）
- 支持多种转化事件：页面访问、注册、下载、付费、自定义
- Last-click 归因模型，归因窗口可配置（1/7/30 天）
- 转化回传方式：Postback URL / Webhook / JS Pixel

### 4.4 AI Agent（MVP-2）

- **扫描**：查询已连接网络中的可用广告位，获取位置、尺寸、受众、定价等信息
- **匹配**：根据产品特征和目标受众，自动评分并选择最合适的广告位
- **物料生成**：调用外部 LLM API（用户自带 Key），生成 banner 图和宣传文案，支持多版本 A/B
- **投放**：自动将广告提交到目标广告位
- **优化**：定期分析投放数据，自动调整策略

### 4.5 去中心化网络（MVP-3）

- **节点发现**：通过公共节点（如 all-mui-ad）发现其他节点，或手动添加
- **节点通信**：节点间通过 REST API 直传广告数据，不经过中间方
- **广告位共享**：每个节点可以声明自己的广告位，供网络中其他节点查询
- **自主性**：每个节点完全自主，可以随时断开连接

### 4.6 积分系统（MVP-4）

| 模式 | 说明 |
|------|------|
| 流量互换 | 展示别人广告赚积分，用积分让别人展示你的广告 |
| CPA 付费 | 按转化计费，每次转化扣减积分 |
| Rev Share | 按收入分成，转化带来付费用户后按比例分成 |
| 积分转账 | 节点间直接互转积分 |

每个节点可以自定义定价。积分在节点间流通。

## 5. 技术架构

### 5.1 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 运行时 | Cloudflare Workers | Serverless，全球边缘，免费额度大 |
| 数据库 | D1 (SQLite) | CF 原生，SQL 灵活 |
| 缓存 | KV | 广告位配置、会话缓存 |
| 文件存储 | R2 | Banner 图片、AI 生成的物料 |
| 消息队列 | Queues | 异步任务（物料生成、归因回传） |
| 语言 | TypeScript (strict) | 全栈 TypeScript |
| 测试 | Vitest | 单元测试 + 集成测试 |
| 格式化 | Biome | lint + format |
| 包管理 | pnpm | |

### 5.2 路由设计

```
/mcp                  MCP Server 端点（SSE transport）
/api/v1/*             REST API（备用，供非 MCP 场景使用）
/widget.js            前端广告渲染脚本
/track/click          点击追踪（重定向 + 计数）
/track/convert        转化回传（Postback）
/peer/*               节点间通信（MVP-3）
```

### 5.3 数据模型

```
products              产品（名称、URL、描述、分类、受众标签）
zones                 广告位（名称、尺寸、位置、所属站点、状态）
ads                   广告（关联产品、类型、物料、状态、权重）
campaigns             广告系列（关联多个广告、预算、时间范围）
impressions           展示记录（zone_id、ad_id、时间、IP hash）
clicks                点击记录（zone_id、ad_id、时间、IP hash、归因参数）
conversions           转化记录（ref、事件类型、时间、来源节点）
nodes                 网络节点（URL、状态、声誉分）
credit_balances       积分余额（节点、余额）
credit_transactions   积分流水（类型、金额、关联记录）
```

## 6. 非功能需求

### 6.1 性能

- 广告渲染请求响应时间 < 50ms（P95）
- 广告渲染脚本异步加载，不阻塞宿主页面
- 利用 KV 缓存广告位配置，减少 D1 查询

### 6.2 安全

- API Key 认证（MCP 和 REST API）
- 归因参数签名防篡改
- IP hash 存储，不记录原始 IP
- 点击和转化的基础防刷（频率限制、异常检测）

### 6.3 隐私

- 不使用第三方 cookie
- 不跨站追踪用户
- 符合 GDPR 基本要求
- 所有数据存储在用户自己的 CF 账户中

### 6.4 可用性

- 单节点无 SLA 要求（独立开发者场景）
- CF Workers 自带全球 CDN 和高可用
- 数据库使用 D1 自动持久化

## 7. MVP 阶段划分

### MVP-0：单实例广告投放 + MCP + 广告渲染

最小可用闭环。一个 MuiAD 实例，能创建广告位、创建广告、在页面上渲染广告、记录展示和点击。

交付标准：AI Agent 通过 MCP 创建广告位和广告，页面能正确渲染广告并计数。

### MVP-1：归因追踪 + 数据统计

在 MVP-0 基础上增加转化追踪能力。广告链接带归因参数，支持 Postback 回传转化事件，提供基础的数据统计 API。

交付标准：能追踪从广告点击到注册/下载/付费的完整链路，通过 MCP 查看推广数据。

### MVP-2：AI Agent

在 MVP-1 基础上增加 AI 自动推广能力。AI 能扫描可用广告位、生成物料、自动投放、持续优化。

交付标准：用户说一句话，AI 自动完成从扫描到投放的全流程。

### MVP-3：节点间通信 + 网络发现

在 MVP-2 基础上增加去中心化网络能力。多个 MuiAD 实例可以互联，共享广告位，互推产品。

交付标准：两个独立部署的 MuiAD 实例能互相发现广告位并提交广告。

### MVP-4：积分系统 + 公共节点

在 MVP-3 基础上增加积分经济。部署 all-mui-ad 公共节点，开发者可以加入网络、赚取和消费积分。

交付标准：积分能在节点间流通，推广效果能通过积分量化。
